## Key Infrastructure Note (Permanent)

**NEVER use pg.Client for DDL in this environment.**

All admin migration routes use `SUPABASE_SERVICE_ROLE_KEY` (JWT) as the PostgreSQL password. Supabase Supavisor (pooler) rejects JWTs — requires the actual DB password. This failure exists in `migrate-p9b`, `migrate-025`, and `migrate-p11-5`. It will not self-resolve.

**DDL workflow:** Write SQL → commit to repo → paste into Supabase SQL Editor at `https://supabase.com/dashboard/project/zyvpoyxdxedcugtdrluc/sql` → click Run. Takes ~2 minutes. Always works.
