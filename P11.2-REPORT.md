## Key Architecture Decisions

### provider_connections Table Design
- **Workspace-scoped** (not project-scoped) — one connection per provider per workspace
- **Partial unique index**: `(workspace_id, provider) WHERE status = 'active'` — multiple pending/expired rows allowed for audit history
- **Token security**: `access_token_ref` stores encrypted refs, not plaintext tokens. P11.2 stores raw (encryption layer in P11.3+)
- **Status FSM**: `pending → active → expired/revoked/error` with documented lifecycle

### Ownership Resolution Algorithm (4-step)
```
1. Query project → workspace_id
2. Look up active provider_connection for (workspace_id, provider)
3. If found + token not expired (buffer: 5 min): return user_managed + token
4. Else: return platform_managed + PAT/GitHub App (existing flow)
   On any error: return platform_managed (never throws)
```

### GitHub: OAuth App (not GitHub App)
- GitHub App (P11.1) is for *provisioning* — creating repos under the platform account
- GitHub OAuth App is for *user connections* — storing user's own GitHub identity/access
- Separate env vars: `GITHUB_CLIENT_ID` / `GITHUB_CLIENT_SECRET` (OAuth App) vs `GITHUB_APP_ID` / `GITHUB_APP_PRIVATE_KEY` (GitHub App)

### Vercel: PAT-based (no OAuth)
- Vercel OAuth is team-plan only; PAT works for all plans
- User pastes PAT → `validateVercelToken()` hits `/v2/user` → store on success
- Token stored in `access_token_ref`, no `refresh_token_ref` needed

### RLS Access Matrix

it('Scenario D — Falls back to platform_managed when token is expired', async () => {
  // Connection exists but token expired 1 hour ago
  mockDb.providerConnections.set('ws-1', [{
    id: 'conn-d',
    workspace_id: 'ws-1',
    provider: 'github',
    status: 'active',
    access_token_ref: 'ghp_expired_token',
    token_expires_at: new Date(Date.now() - 3600_000).toISOString(), // 1h ago
  }])

  const result = await resolveProviderOwnership({
    projectId: 'proj-d', provider: 'github', supabase: mockSupabase
  })

  expect(result.mode).toBe('platform_managed')
  expect(result.token).toBeUndefined()
  // provision/route.ts uses platform PAT/GitHub App token (existing flow)
})
